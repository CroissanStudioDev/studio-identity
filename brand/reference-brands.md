# Reference brands — what we're aiming at

Studios and products whose work calibrates the **quality bar** for Croissan output.
Not role models for visual style (we're already opinionated about that), but for **craft
discipline** — the thing that separates "looks like a thoughtful studio" from "looks
like a fast template."

When a contributor or agent is unsure whether an output reads as ours, this list is the
"is the level high enough?" check. If you can imagine the work sitting next to any of
these without embarrassment, it's good.

## The list

## How to use this list

- **You're a contributor and unsure.** Pick the closest reference; ask "would this fit
  there?" If no, raise the bar.
- **You're an AI agent generating output.** Pull this list before generating a
  marketing page or pitch deck. Pattern-match against the closest reference, **not** the
  generic AI-SaaS template.
- **You're reviewing.** If a generated artifact reads like Vercel/Linear/Rauno-tier,
  ship. If it reads like a generic Tailwind landing template, send back.

## Anti-references — what we explicitly aren't

We do **not** want our output to read like:

- Generic Y Combinator launch landings (Hero → Features → Pricing → CTA template)
- Indie SaaS template shops (boilerplate, no editorial)
- "AI-powered" startup landing kits (gradient mesh hero, "trusted by 10,000+", emoji bullets)
- Corporate enterprise sites (stock handshakes, "synergy", three-column feature cards)

If a generated page reads like any of these, the [`agents/anti-ai-slop.md`](../agents/anti-ai-slop.md)
review will catch it.

## When to refresh this list

A reference becomes stale when its work no longer represents the bar. Refresh when:

- A site on the list pivots to a generic look (it happens — sites rebrand badly).
- A new studio or product clearly out-disciplines the list. Add it; remove the weakest
  current entry. Don't accumulate more than ~7 entries; the point is calibration, not
  inspiration.
